# Compare Legacy Skill

레거시 기능과 WPF 구현을 비교하여 누락된 로직을 찾습니다.

## 사용법
- `/compare-legacy <기능명>` - 기능별 비교
- `/compare-legacy <레거시파일> <WPF파일>` - 파일 직접 비교

## 비교 항목

### 1. 메서드 커버리지
| 레거시 메서드 | WPF 대응 | 상태 |
|--------------|---------|------|
| WorkerSearch() | SearchAsync() | 완료 |
| WorkerSave() | - | 누락 |

### 2. 비즈니스 로직
- 조건문/분기 로직
- 유효성 검증
- 에러 처리

### 3. DB 쿼리
- SELECT/INSERT/UPDATE/DELETE 커버리지
- 파라미터 매핑

### 4. 외부 연동
- API 호출
- 장비 명령

## 출력 형식

```markdown
# [기능명] 레거시 vs WPF 비교

## 커버리지 요약
- 메서드: 8/10 (80%)
- DB 쿼리: 5/5 (100%)
- API: 3/4 (75%)

## 누락 항목
| 항목 | 레거시 위치 | 설명 | 우선순위 |
|------|------------|------|---------|
| WorkerDelete | Worker.cs:250 | 근로자 삭제 | 높음 |

## 구현 권장사항
1. ...
```
