# 프리마켓 브리핑 페이지 v2 — 통합 카드 재구성

- **작성일**: 2026-04-26 (KST)
- **대상 파일**: `api/templates/briefing.html`
- **브랜치**: `dev`
- **연계 문서**: `_docs/20260425101355_pre_market_briefing.md` (브리핑 파이프라인 운영 매뉴얼)

## 1. 배경

현재 프리마켓 브리핑 페이지는 다음 3개 섹션이 평면적으로 나열돼 있다:

1. **상단 헤드라인**(`us.headline`) — 한 줄 임팩트 ("📰 INTC +23.60%...")
2. **① 오늘 S&P500 핵심 상승 섹터 & 종목** (`us.groups[]`)
3. **② 코스피/코스닥 섹터별 수혜 시나리오** (`kr_impact[]`)
4. **③ 모닝 코멘트** (`bd.morning_brief`)

데이터상 ①과 ②는 `sector_norm`(예: `semiconductors`, `it_software`) 공통키로 본질적으로 같은 섹터를 두 번 표현하고 있으나, UI에서 시각적 연결이 없다. 또한 미국 종목 pill은 ticker만 노출돼 한국 사용자가 종목을 즉시 식별하기 어렵다(예: `INTC`, `ARM`, `QCOM`).

## 2. 목표

| 사용자 요구 | 해결 방식 |
|---|---|
| 미국 종목명 풀네임 확인 | pill 에 `title="{asset_name}"` 툴팁 + 클릭 시 종목 상세 페이지(`/pages/stocks/{ticker}`) 이동 |
| US 섹터 ↔ KR 시나리오 매칭 | `sector_norm` 키로 outer-join → 섹터당 1개 통합 카드(섹터 단위 = 카드 단위) |
| KR 시나리오 접기/펴기 | `<details>`/`<summary>` 네이티브 사용. KR picks ≥ 1 인 카드는 기본 펼침, 그 외 접힘 |
| 모닝 코멘트와 헤드라인 통합 | 페이지 최상단에 `us.headline` + `bd.morning_brief` 단일 그라디언트 카드(hero). ③ 섹션 제거 |

## 3. 비범위(이번에 안 함)

- `analyzer/briefing_main.py`, `analyzer/prompts.py` (LLM 프롬프트·JSON 스키마)
- `pre_market_briefings` DB 마이그레이션 (스키마 동일)
- 시장 레짐 디버그 섹션 리디자인 (그대로 유지)
- 새 시각화·차트 추가
- 모바일 전용 패턴 별도 도입 (기존 미디어 쿼리 패턴 재사용)

## 4. 데이터 구조 (변경 없음)

브리핑 페이지가 받는 `briefing` 객체는 **두 개의 us_summary**가 있다 (헷갈리니 명확히):

### 4.1 `briefing.us_summary` — Raw 집계 (`analyzer.overnight_us.compute_us_overnight_summary`)
```jsonc
{
  "trade_date": "2026-04-24",
  "universe_size": 600,
  "top_movers": [
    { "ticker": "INTC", "asset_name": "Intel Corporation",
      "market": "NASDAQ", "sector_norm": "semiconductors", "change_pct": 23.60 },
    ...                                                  // universe-wide top, 섹터 무관
  ],
  "top_losers": [...],
  "sector_aggregates": [
    {
      "sector_norm": "semiconductors",
      "label": "반도체",
      "n": 30,
      "avg_change_pct": 4.66,          // ← 평균 등락 출처(고정)
      "median_change_pct": 3.11,
      "max_change_pct": 23.60,
      "min_change_pct": -1.50,
      "top_stocks": [                  // ← 섹터별 ticker→asset_name 매핑 가능
        { "ticker": "INTC", "asset_name": "Intel Corporation", "market": "NASDAQ",
          "sector_norm": "semiconductors", "change_pct": 23.60 },
        ...
      ]
    },
    ...
  ],
  "indices": { "SP500": {...}, "NDX100": {...} }
}
```

### 4.2 `briefing.briefing_data` — LLM 출력
```jsonc
{
  "us_summary": {                                        // ← 템플릿의 `us` 변수
    "headline": "INTC +23.60% 이례적 급등이...",
    "groups": [
      {
        "sector_norm": "semiconductors",                 // ← 매칭 키
        "label": "반도체 (SEMICONDUCTORS)",
        "top_movers": [
          { "ticker": "INTC", "change_pct": 23.60 },     // ⚠ asset_name 없음
          { "ticker": "ARM",  "change_pct": 14.76 },
          ...
        ],
        "catalyst": "INTC 단일 종목 +23.60% 급등..."
      },
      ...
    ]
  },
  "kr_impact": [
    {
      "sector_norm": "semiconductors",                   // ← 같은 키로 매칭
      "label": "반도체 장비·메모리",
      "strength": "gap_up_strong",                       // gap_up_strong | gap_up_likely | upside_expected | partial_benefit | limited
      "korean_picks": [
        { "ticker": "000660", "asset_name": "SK하이닉스", "market": "KOSPI",
          "rationale": "HBM3E·HBM4 공급 과점...",
          "expected_open_change_pct": "+2~4%" },
        ...
      ],
      "catalysts_kr": "...",
      "related_etfs": ["KODEX 반도체", ...]
    },
    ...
  ],
  "morning_brief": "...서술형 코멘트..."
}
```

### 4.3 핵심 결과
- **평균 등락 출처**: `briefing.us_summary.sector_aggregates[].avg_change_pct` (sector_norm 키로 lookup)
- **asset_name 풀네임 출처**: LLM groups의 top_movers엔 없음 → `briefing.us_summary` 의 `top_movers[]` ∪ `sector_aggregates[].top_stocks[]` 를 ticker→asset_name dict로 합쳐 lookup. 미스 시 ticker 자체를 폴백.
- **market 출처**: 같은 dict에 있음 — pill 의 `?market=NASDAQ` 쿼리스트링용.

> 라우트 확인: `GET /pages/stocks/{ticker}?market={market}` → `stock_cockpit.html` (`api/routes/stocks.py:561`)

### 5.2 통합 카드 본문 구성 규칙

| 섹터 분류 | 표시 |
|---|---|
| **US ∩ KR** (양쪽 다 있음) | US movers + KR picks 둘 다 표시. 기본 **펼침**. |
| **US only** (KR 매칭 없음) | US movers + `🇰🇷 한국 매칭 없음 · 영향 제한적` 한 줄. 기본 **접힘**. |
| **KR only** (US groups 에 없음) | 카드 후미에 KR picks 만. 기본 **펼침**. (드물지만 가능) |

### 5.3 헤더(summary) 구성

```
[icon] [라벨]              [평균 등락 %]   [KR strength badge or "KR 매칭 없음"]   [▼/▶]
```

- **icon**: loop.index 기반 이모지 매핑 유지 (🔵⚡🍎💊📈)
- **평균 등락**: `top_movers` 의 `change_pct` 평균을 템플릿 측에서 계산하거나, `us_summary.sector_aggregates` 에서 같은 `sector_norm` row 의 `mean_change_pct` 를 dict 변환해 lookup. 후자가 정확.
- **KR strength badge**: 기존 `.strength-{gap_up_strong|...}` 색상 매핑 그대로
- **▼/▶**: `<details>` 네이티브 토글 인디케이터를 CSS 로 회전

## 6. 매칭 로직 (Jinja2)

`do` 익스텐션 의존을 피하고 `namespace()` 패턴 사용:

```jinja2
{# 1) kr_impact 를 sector_norm 키 dict 로 #}
{% set kr_map = namespace(d={}) %}
{% for k in kr %}
  {% set _ = kr_map.d.update({k.sector_norm: k}) %}
{% endfor %}

{# 2) sector_aggregates 를 sector_norm 키 dict 로 (avg_change_pct 조회용) #}
{% set agg_map = namespace(d={}) %}
{% for s in (briefing.us_summary or {}).get('sector_aggregates', []) %}
  {% set _ = agg_map.d.update({s.sector_norm: s}) %}
{% endfor %}

{# 3) ticker → {asset_name, market} 매핑 dict — pill 풀네임 + 링크용 #}
{% set name_map = namespace(d={}) %}
{% for r in (briefing.us_summary or {}).get('top_movers', []) %}
  {% set _ = name_map.d.update({r.ticker: {'asset_name': r.asset_name, 'market': r.market}}) %}
{% endfor %}
{% for r in (briefing.us_summary or {}).get('top_losers', []) %}
  {% set _ = name_map.d.update({r.ticker: {'asset_name': r.asset_name, 'market': r.market}}) %}
{% endfor %}
{% for s in (briefing.us_summary or {}).get('sector_aggregates', []) %}
  {% for r in s.get('top_stocks', []) %}
    {% set _ = name_map.d.update({r.ticker: {'asset_name': r.asset_name, 'market': r.market}}) %}
  {% endfor %}
{% endfor %}

{# 4) US 섹터 순회 — 통합 카드 렌더 #}
{% for g in us.groups %}
  {% set kr_grp = kr_map.d.get(g.sector_norm) %}
  {% set agg = agg_map.d.get(g.sector_norm) %}
  {% set has_kr = kr_grp and (kr_grp.korean_picks | length) > 0 %}
  <details class="sector-card" {% if has_kr %}open{% endif %}>
    <summary class="sector-summary">
      <span class="sector-icon">{{ icon }}</span>
      <span class="sector-label">{{ g.label or g.sector_norm }}</span>
      {% if agg and agg.avg_change_pct is not none %}
        <span class="sector-mean {{ 'up' if agg.avg_change_pct>=0 else 'dn' }}">
          평균 {{ '%+.2f' % agg.avg_change_pct }}%
        </span>
      {% endif %}
      {% if kr_grp and kr_grp.strength %}
        <span class="strength-badge strength-{{ kr_grp.strength }}">
          {{ STRENGTH_LABEL[kr_grp.strength] }}
        </span>
      {% else %}
        <span class="strength-badge strength-none">KR 매칭 없음</span>
      {% endif %}
    </summary>
    {# 본문 — US movers, catalyst, KR picks ... — name_map 사용 #}
  </details>
{% endfor %}

{# 5) KR-only 섹터 fallback #}
{% set us_keys = us.groups | map(attribute='sector_norm') | list %}
{% for k in kr if k.sector_norm not in us_keys %}
  ...
{% endfor %}
```

`STRENGTH_LABEL` 은 템플릿 상단에서 `{% set STRENGTH_LABEL = {...} %}` 로 정의(현재 inline if/elif 블록을 dict 로 정리).

**평균 등락 출처(고정 규칙)**: `agg_map.d.get(g.sector_norm).avg_change_pct` 우선. 없으면 `g.top_movers` 의 `change_pct` 평균을 Jinja2 측에서 직접 계산해 사용. 둘 다 비면 헤더에서 평균등락 표기 자체를 생략한다.

## 7. US Movers Pill — 풀네임 + 링크

기존:
```jinja2
<span class="mover-pill">
  <strong>{{ m.ticker }}</strong>
  <span>{{ '%+.2f' % m.change_pct }}%</span>
</span>
```

신규 (LLM groups의 mover에는 asset_name이 없으므로 `name_map`에서 lookup):
```jinja2
{% set info = name_map.d.get(m.ticker, {}) %}
{% set fullname = info.asset_name or m.ticker %}
{% set mkt = info.market or '' %}
<a class="mover-pill"
   href="/pages/stocks/{{ m.ticker }}{% if mkt %}?market={{ mkt }}{% endif %}"
   title="{{ fullname }}{% if mkt %} · {{ mkt }}{% endif %}">
  <strong>{{ m.ticker }}</strong>
  <span class="{{ 'mover-pos' if (m.change_pct or 0) >= 0 else 'mover-neg' }}">
    {{ '%+.2f' % (m.change_pct or 0) }}%
  </span>
</a>
```

- `<span>` → `<a>` 로 교체. 기존 pill 스타일 유지(`text-decoration: none; color: inherit;` 추가).
- hover 시 `title` 로 풀네임 즉시 노출 (네이티브 툴팁).
- `name_map` lookup 미스 시 ticker 폴백 — 링크는 market 쿼리스트링 없이 `/pages/stocks/{ticker}` 로만 이동(라우트가 universe에서 market 자동 조회).

## 8. Hero 카드 (헤드라인 + 모닝 코멘트)

기존 `.brief-headline` (그라디언트 박스, headline 한 줄) 과 ③ `.brief-morning` (좌측 accent bar, morning_brief) 을 한 컨테이너로 합친다.

```jinja2
{% if us.get('headline') or bd.get('morning_brief') %}
<section class="brief-hero">
  {% if us.headline %}
  <div class="brief-hero-headline">📰 <strong>{{ us.headline }}</strong></div>
  {% endif %}
  {% if bd.morning_brief %}
  <div class="brief-hero-body">{{ bd.morning_brief }}</div>
  {% endif %}
</section>
{% endif %}
```

스타일:
- 그라디언트 배경(기존 `.brief-headline`)
- 헤드라인 폰트 강조 (`font-size: 16px; font-weight: 600;`)
- body 는 `line-height: 1.65; font-size: 14px; color: var(--text);` — 기존 `.brief-morning` 색감 유지하되 좌측 accent bar 는 제거(중복 강조 방지)
- 둘 사이 hairline divider (`border-top: 1px dashed rgba(255,255,255,0.08); margin: 12px 0;`)

## 11. 리스크 & 완화

| 리스크 | 완화 |
|---|---|
| `sector_aggregates` 가 `mean_change_pct` 를 안 갖고 있을 가능성 | 구현 첫 단계에서 `analyzer/overnight_us.py:compute_us_overnight_summary` 반환값 확인. 없으면 `top_movers` 평균을 템플릿에서 계산. |
| KR-only 섹터 발생 빈도 미상 (실제로 거의 없음) | fallback 루프는 두되, 데이터가 없으면 자연스럽게 안 그려지도록 if-guard. |
| `<details>` 의 ▶/▼ 인디케이터가 OS/브라우저 기본 마커와 충돌 | `summary { list-style: none; }` + `summary::-webkit-details-marker { display: none; }` 로 기본 마커 죽이고 커스텀 회전 화살표 사용 |
| `asset_name` 이 영문(`asset_name_en`)일 가능성 | `overnight_us.py:250` 가 이미 `asset_name_en` 우선 → `asset_name` 폴백 처리. 한국 사용자에겐 영문이 오히려 나을 수 있음(인텔 vs Intel — Intel 이 더 친숙). 그대로 사용. |
| pill 이 `<a>` 가 되면 키보드 포커스 흐름 영향 | 기존 페이지엔 인터랙티브 요소가 KR pick 영역에만 있어 영향 미미. tab 순서 자연스러움. |
| 카드 본문에 US movers 가 많은 경우 모바일에서 가로 스크롤 | 기존 `flex-wrap: wrap` 유지로 자동 줄바꿈. 추가 작업 없음. |
