## 4. 의존성 & 병렬화 가이드

### 4.1 크리티컬 패스 (Phase 0)

```
외부 리드타임 트랙 (착수 즉시 병렬, 장기)
  A-019 (법규 신고) ────────────────────────────────► A-021 (약관)
  A-023 (PG 심사)   ──────► A-004 (구독 BE) ──► A-014 (구독 UI) ──► 결제 검증
                            ▲
  A-003 (tier 스키마) ──────┘

UI/UX 트랙
  A-001 (트랙레코드 BE) ──► A-013 (트랙레코드 UI)
  A-002 (한도 미들웨어) ──► A-015 (업그레이드 UX)
  A-020 (면책 문구)     ──► A-016 (면책 삽입)

베타 오픈 게이트
  A-018 (베타 랜딩) ──► A-027 (베타 초대)
```

### 4.2 병렬화 규칙

- **동일 Stage 내 병렬**: 같은 Stage 카드의 Action Item은 의존성만 없으면 동시 착수.
- **Stage 건너뛰기 금지**: 다음 Stage의 Exit Gate는 이전 Stage Gate 통과를 전제한다.
- **장기 트랙 조기 착수**: A-019·A-023·A-007·A-024는 리드타임이 길므로 해당 Stage 시작 즉시 외부 프로세스를 개시한다.
- **백엔드 → 프론트 순서**: 각 기능은 Backend 완료(또는 Mock API 합의) 후 Frontend 착수.
