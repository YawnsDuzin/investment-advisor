## 우선순위: 높음

- [x] **조회 API 보호** — `/sessions`, `/themes`, `/proposals` JSON API에 `get_current_user_required` Depends 추가 완료. AUTH_ENABLED=true일 때 로그인 필수, false일 때 기존 동작 유지
- [ ] **User 역할 존재 의미 재정의** — 현재 User는 조회만 가능하고 채팅 불가. 비인증과 거의 동일한 경험
  - 안 A: User에게도 채팅 허용 (일일 메시지 수 제한, 예: 10회/일)
  - 안 B: User를 "승인 대기" 역할로 활용 — Admin/Moderator 승인 후 승격
  - 안 C: User에게 "관심 종목 북마크" 등 조회 + 경량 기능 제공
- [x] **비밀번호 변경 기능** — `/auth/change-password` POST 엔드포인트 + `/pages/profile` 프로필 페이지 구현 완료. 현재 비밀번호 검증 → 새 비밀번호 변경

## 우선순위: 중간

- [ ] **개인화 기능** — 인증 기반 사용자별 맞춤 기능
  - 관심 종목 워치리스트: 사용자별 관심 티커 저장, 대시보드에서 하이라이트
  - 알림 구독: 특정 테마/종목이 분석에 등장 시 표시
  - 분석 결과 메모: 제안(proposal)에 개인 코멘트 남기기
- [ ] **활동 로그 / 감사 추적** — 현재 채팅만 user_id 기록, 나머지 조회 활동 미추적. `user_activity_log` 테이블 추가하여 누가 언제 어떤 분석/테마/종목을 조회했는지 기록. Admin 대시보드에서 사용 패턴 파악
- [x] **세션 만료 UX** — Access Token 만료 시 채팅 중 메시지 전송이 401로 실패. 프론트엔드에서 401 감지 시 자동 `/auth/refresh` 호출, 실패 시 로그인 페이지 리다이렉트하는 JS 인터셉터 추가

## 우선순위: 낮음

- [ ] **Moderator 역할 활용도 확대** — 현재 채팅 + 사용자 관리(제한적)만 가능
  - 뉴스 번역 실행 권한 부여 (분석 실행은 Admin만)
  - 테마/종목에 "편집자 노트" 추가 (공개 코멘트)
  - 분석 결과 품질 평가 (피드백 기록)
