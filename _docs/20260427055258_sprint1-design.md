# Sprint 1 — 통합 Design

> 작성: 2026-04-27 KST · 작성자: Claude Code (Opus 4.7) · 승인 대기
> 관련 프롬프트: [_docs/_prompts/20260427_prompt.md](_prompts/20260427_prompt.md)

## Executive Summary

투자 분석 시스템에 *동종 서비스 대비 차별화*가 큰 5개 기능을 한 스프린트(5-7주, 일부 PR 병렬화)로 묶어 출시한다.

1. **NL → SQL 종목 검색 (E)** — 자연어 질의를 안전 가드된 SELECT로 변환, read-only DB user로 실행
2. **Bull/Bear Red Team (D)** — Stage 2 종목 분석을 강세·약세 페르소나 두 번 + 종합으로 깊이 강화 (Premium·기본 OFF)
3. **스크리너 UI (6)** — `screener_presets` v40 인프라 위에 거장 시드 5 + 운영 자동 5 + 사용자 커스텀
4. **글로벌 뉴스 번역 (J)** — 일/중/유럽/미국 RSS + Haiku 번역, region별 카테고리 분리로 Stage 1 입력 풍부화
5. **차트 Vision 실험 (G)** — 사용자 차트 이미지 → Claude vision → 패턴 식별 + 카운터 시나리오. 익명 1회 체험

DB 변경은 단일 마이그레이션 v41로 통합. 7개 PR로 분할(각 ~1주). 라즈베리파이 24/7 운영 전제.

- 라즈베리파이 4 (8GB) 24/7 운영 가능 — 메모리·CPU 누적 방지

## 3. 데이터 모델 — 마이그레이션 v41

`shared/db.py` 의 `SCHEMA_VERSION = 41` + `_migrate_to_v41()` 추가. 단일 트랜잭션 내 멱등 실행.

```sql
-- (1) NL → SQL 검색 이력 / rate-limit 카운팅 / 학습 자료
CREATE TABLE IF NOT EXISTS nl_search_history (
  id            BIGSERIAL PRIMARY KEY,
  user_id       INT REFERENCES users(id) ON DELETE CASCADE,
  query_text    TEXT NOT NULL,
  generated_sql TEXT,
  result_count  INT,
  status        TEXT NOT NULL CHECK (status IN ('ok','rejected','timeout','error')),
  reject_reason TEXT,           -- guard 차단 이유 (e.g. 'WRITE_NOT_ALLOWED')
  duration_ms   INT,
  created_at    TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX IF NOT EXISTS nl_search_history_user_date
  ON nl_search_history(user_id, created_at DESC);

-- (2) Stage 2 Red Team 결과 — 기존 stock_analyses 확장
ALTER TABLE stock_analyses
  ADD COLUMN IF NOT EXISTS bull_view         JSONB,
  ADD COLUMN IF NOT EXISTS bear_view         JSONB,
  ADD COLUMN IF NOT EXISTS synthesis_summary TEXT,
  ADD COLUMN IF NOT EXISTS red_team_enabled  BOOLEAN DEFAULT FALSE;

-- (3) 글로벌 뉴스 다국어 — 기존 news_articles 확장
ALTER TABLE news_articles
  ADD COLUMN IF NOT EXISTS lang           VARCHAR(8)  DEFAULT 'ko',
  ADD COLUMN IF NOT EXISTS title_original TEXT,
  ADD COLUMN IF NOT EXISTS region         VARCHAR(16);  -- KR|US|JP|CN|EU|GLOBAL

-- (4) 차트 Vision 로그 — 이미지 자체는 미저장, 응답·해시만 보관
CREATE TABLE IF NOT EXISTS chart_vision_log (
  id              BIGSERIAL PRIMARY KEY,
  user_id         INT REFERENCES users(id) ON DELETE CASCADE,  -- NULL = 익명
  anonymous_token CHAR(64),                                     -- sha256(session_id||ip||UA)
  image_hash      CHAR(64) NOT NULL,
  ticker          VARCHAR(16),
  response_text   TEXT,
  duration_ms     INT,
  created_at      TIMESTAMPTZ DEFAULT NOW(),
  CHECK (user_id IS NOT NULL OR anonymous_token IS NOT NULL)
);
CREATE INDEX IF NOT EXISTS chart_vision_log_user_date
  ON chart_vision_log(user_id, created_at DESC) WHERE user_id IS NOT NULL;
CREATE INDEX IF NOT EXISTS chart_vision_log_anon_date
  ON chart_vision_log(anonymous_token, created_at DESC) WHERE anonymous_token IS NOT NULL;

-- (5) KRX 투자자 수급 일별 사전 집계 — 외국인 연속 매수 시드 프리셋용 (§7.2.1)
CREATE TABLE IF NOT EXISTS krx_investor_flow_daily (
  ticker         VARCHAR(16) NOT NULL,
  trade_date     DATE NOT NULL,
  foreign_net    BIGINT,
  inst_net       BIGINT,
  foreign_streak INT,        -- 양수=연속순매수, 음수=연속순매도
  PRIMARY KEY (ticker, trade_date)
);
CREATE INDEX IF NOT EXISTS idx_kif_streak
  ON krx_investor_flow_daily(trade_date, foreign_streak DESC);

-- (6) Universe 일배치 factor percentile — 모멘텀·O'Neil 시드 (§7.2.2)
CREATE TABLE IF NOT EXISTS stock_universe_factor_snapshot (
  ticker        VARCHAR(16) NOT NULL,
  market        VARCHAR(8)  NOT NULL,
  snapshot_date DATE NOT NULL,
  r1m_pctile    NUMERIC(5,4),
  r3m_pctile    NUMERIC(5,4),
  r6m_pctile    NUMERIC(5,4),
  r1y_pctile    NUMERIC(5,4),
  vol60_pctile  NUMERIC(5,4),
  PRIMARY KEY (ticker, market, snapshot_date)
);
CREATE INDEX IF NOT EXISTS idx_factor_latest
  ON stock_universe_factor_snapshot(ticker, market, snapshot_date DESC);

-- (7) screener_presets 시드 INSERT (v40 에서 컬럼만 준비, v41 에서 시드 본격 INSERT)
-- 거장 5: Buffett / Lynch / Graham / O'Neil / Greenblatt (초기 컷 단순화 — §7.2 표 참조)
-- 운영 자동 5: 52주 신고가 / 거래량 급증 / 외국인 5일 연속 / 모멘텀 강세 / 저밸류+고배당
-- ON CONFLICT (strategy_key) WHERE is_seed=TRUE DO UPDATE  — 멱등 UPSERT
```

**필드 노트:**
- `nl_search_history.reject_reason` — guard 차단 사유를 enum 형 문자열로 저장 (사후 분석·rate-limit 정확도용).
- `chart_vision_log.anonymous_token` — 익명 1회 체험 추적용. `sha256(session_cookie || client_ip || User-Agent)` 32바이트 hex. 부분 인덱스로 anonymous 카운트 빠르게.
- `news_articles.lang/region` 은 NULLABLE이지만 default `'ko'`. 기존 row 는 자동으로 ko/KR 으로 표시(`region` 은 NULL 유지 — 하위호환).
- `stock_analyses.red_team_enabled` 는 default FALSE — 마이그레이션 직후 기존 row 표시 변화 없음.

## 5. Feature E — NL → SQL 종목 검색

### 5.1 흐름

```
[사용자 NL 질의]
   └─► POST /api/screener/nl-search  { query }
       │
       ├─ tier·rate limit (nl_search_history 일일 카운트)
       │
       ├─ Claude SDK query()
       │     model:  MODEL_ANALYSIS  (claude-sonnet-4-6)
       │     system: 스키마 카탈로그 + "Single SELECT only. PostgreSQL."
       │     user:   질의 + few-shot 3개
       │     출력:   {sql, explanation}  (JSON)
       │
       ├─ shared/nl_sql_guard.py 검증
       │     • sqlparse AST 단일 statement
       │     • SELECT 만
       │     • 화이트리스트 테이블만
       │     • LIMIT 누락 → LIMIT 100 강제 주입
       │     • 금지 키워드 차단
       │
       ├─ NL_SEARCH_READONLY_DSN 으로 실행
       │     (psycopg2 connect → SET statement_timeout='10s' → execute → fetchmany)
       │
       └─► { sql, explanation, rows: [...], result_count, duration_ms }
            └─ nl_search_history 기록 (rejected 도 기록)
```

### 5.2 SQL 가드 ([shared/nl_sql_guard.py](../shared/nl_sql_guard.py) 신규)

```python
ALLOWED_TABLES = frozenset({
    "stock_universe",
    "stock_universe_ohlcv",
    "stock_universe_fundamentals",
    "investment_proposals",
    "theme_tracking",
    "proposal_tracking",
    "investment_themes",          # 테마명 검색용
    "analysis_sessions",          # 날짜 필터용
})
FORBIDDEN_KEYWORDS = frozenset({
    "INSERT", "UPDATE", "DELETE", "DROP", "ALTER", "CREATE",
    "GRANT", "REVOKE", "COPY", "EXECUTE", "CALL", "TRUNCATE",
    "VACUUM", "ANALYZE", "REINDEX", "PG_", "SET ", "RESET ",
})

def validate_and_rewrite(sql: str, *, max_limit: int = 100) -> tuple[str, str | None]:
    """returns (safe_sql, reject_reason). reject_reason None == ok."""
    # 1) sqlparse — 단일 statement 강제
    # 2) DML/DDL 키워드 토큰 단위 검사 (주석·문자열 우회 방지: stripped tokens 만)
    # 3) FROM/JOIN 식별자 추출 → ALLOWED_TABLES 부분집합 검사
    # 4) LIMIT 절 없으면 INSERT
    # 5) ; 문자 1개 초과 reject
```

**가드 통과 못 하면 SDK 호출 결과를 노출하지 않고 reject_reason 만 사용자에 표시** — Claude 응답이 잘못되어도 안전.

### 5.3 read-only DB user (운영기 사전 작업)

```sql
-- PostgreSQL 운영기 1회 실행
CREATE ROLE nl_reader LOGIN PASSWORD 'CHANGE_ME_STRONG';
GRANT CONNECT ON DATABASE investment_advisor TO nl_reader;
GRANT USAGE ON SCHEMA public TO nl_reader;
GRANT SELECT ON ALL TABLES IN SCHEMA public TO nl_reader;
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT SELECT ON TABLES TO nl_reader;
ALTER ROLE nl_reader SET statement_timeout = '10s';
```

`.env` 에 `NL_SEARCH_READONLY_DSN=postgresql://nl_reader:STRONG@localhost:5432/investment_advisor` 추가. 미설정 시 main DSN 사용 + 프로덕션 경고 로그(`logger.warning("NL_SEARCH_READONLY_DSN not set — using main DSN")`).

### 5.5 system prompt 골격

```
You are a SQL generator. Translate the Korean/English natural-language
query into a single PostgreSQL SELECT statement.

Available tables (with key columns):
  stock_universe(ticker, market, asset_name, sector_norm, sector_krx,
                 industry, market_cap, listed)
  stock_universe_ohlcv(ticker, market, trade_date, open, high, low,
                       close, volume, change_pct)
  stock_universe_fundamentals(ticker, market, snapshot_date, per, pbr,
                              eps, bps, dps, dividend_yield, roe)
  investment_proposals(ticker, theme_id, action, conviction,
                       discovery_type, foreign_net_buy_signal,
                       return_1m_pct, return_3m_pct, post_return_3m_pct, ...)
  ...

Rules:
  • Output JSON: {"sql": "...", "explanation": "..."}
  • Single SELECT statement only — no semicolons.
  • Use ILIKE for text search (Korean & English).
  • Always include LIMIT (max 100).
  • Use NUMERIC casts for percentile/threshold comparisons.
  • Prefer recent data (e.g., latest snapshot_date per ticker).

Few-shot examples:
  Q: 외국인이 5일 연속 매수한 반도체 종목
  A: { "sql": "SELECT u.ticker, u.asset_name FROM stock_universe u
        JOIN investment_proposals p USING (ticker)
        WHERE u.sector_norm = 'semiconductors'
          AND p.foreign_net_buy_signal = 'consecutive_5d'
        LIMIT 100",
       "explanation": "..." }
  ...
```

## 6. Feature D — Bull/Bear Red Team

### 6.1 흐름

```python
# analyzer/analyzer.py — stage2_analyze_stock() 내부
async def stage2_analyze_stock(...):
    if cfg.enable_red_team and user_tier in cfg.red_team_tiers:
        bull, bear = await asyncio.gather(
            _query_stage2(STAGE2_BULL_SYSTEM, prompt),
            _query_stage2(STAGE2_BEAR_SYSTEM, prompt),
        )
        synthesis = await _query_stage2(
            STAGE2_SYNTHESIS_SYSTEM,
            f"BULL_VIEW:\n{json.dumps(bull, ensure_ascii=False)}\n\n"
            f"BEAR_VIEW:\n{json.dumps(bear, ensure_ascii=False)}",
        )
        return {
            "bull_view": bull,
            "bear_view": bear,
            "synthesis_summary": synthesis["summary"],
            "red_team_enabled": True,
            **synthesis,   # decisive_factors, judgment_triggers 등
        }
    else:
        # 기존 단일 Stage 2 — 변경 없음
        return await _query_stage2(STAGE2_SYSTEM, prompt)
```

### 9.2 익명 1회 추적

```python
def _anon_token(request: Request) -> str:
    sess = request.cookies.get("anon_session")
    if not sess:
        sess = secrets.token_hex(32)
        # 응답에서 set_cookie (max_age=365d, http_only, secure if prod)
    ip  = request.client.host
    ua  = request.headers.get("user-agent", "")
    return hashlib.sha256(f"{sess}|{ip}|{ua}".encode()).hexdigest()
```

쿠키 + IP + UA 조합으로 익명 사용자 식별. 프라이버시 — 원본 IP·UA 는 저장 안 함, hash 만.

## 12. 주요 위험 · 완화

| 위험 | 영향 | 완화 |
|---|---|---|
| NL → SQL injection (Claude 응답 또는 사용자 우회) | 데이터 유출 / 변조 | (1) 별도 read-only DB user (2) sqlparse 화이트리스트 (3) statement_timeout 10s — 3중 방어 |
| 프롬프트 인젝션으로 SQL 가드 우회 시도 | 비허용 SQL 생성 | 가드는 SDK 응답 SQL 에 적용 — Claude 가 악성이어도 차단 |
| Red Team 토큰 비용 폭발 | Claude 구독 cap 초과 | Premium 한정 + env 기본 OFF + on-demand 클릭 트리거 |
| 글로벌 RSS rate-limit / 차단 | 뉴스 수집 실패 | 기존 httpx UA·헤더 유지, MAX_ARTICLES_PER_FEED 동일, 실패 region 은 skip 후 로그 |
| Vision 멀웨어 업로드 | 서버 보안 | magic byte 검증 + 메모리 처리 + 디스크 저장 X + Pillow 디코드 검증 |
| 라즈베리파이 메모리 누수 | OOM 으로 service 중단 | 모든 신규 호출 동기 단위로 끊어 누적 방지 + Vision 이미지 ≤5MB 한도 |
| 익명 Vision 추적 우회 (쿠키 삭제 + VPN) | 무한 익명 사용 | 비용 영향 미미 (이미지당 토큰 작음) — 광고비로 간주. Pro+ 가 본 가치 |
| 마이그레이션 v41 부분 실패 | 일부 컬럼만 추가됨 | 단일 트랜잭션 + `IF NOT EXISTS` 멱등 — 재실행 시 자동 복구 |

### PR-5 배포 (NL → SQL)

```sql
-- 운영기 PostgreSQL
CREATE ROLE nl_reader LOGIN PASSWORD 'STRONG_PASSWORD';
GRANT CONNECT ON DATABASE investment_advisor TO nl_reader;
GRANT USAGE ON SCHEMA public TO nl_reader;
GRANT SELECT ON ALL TABLES IN SCHEMA public TO nl_reader;
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT SELECT ON TABLES TO nl_reader;
ALTER ROLE nl_reader SET statement_timeout = '10s';
```

`.env` 갱신:
```
NL_SEARCH_READONLY_DSN=postgresql://nl_reader:STRONG@localhost:5432/investment_advisor

NL_SEARCH_TIMEOUT_SEC=10
NL_SEARCH_RESULT_LIMIT=100
```
