## 작업 지시

아래 관점에서 대시보드 개선안을 제시해주세요:

### 1. 정보 계층 구조 (Information Hierarchy) 재설계
- 현재 9개 섹션이 단순 나열되어 있음. 투자자가 "오늘 뭘 봐야 하는지" 빠르게 파악할 수 있도록 정보 우선순위를 재배치
- 각 섹션의 유지/통합/이동/삭제 여부와 그 이유를 명시
- "First Glance → Deep Dive" 패턴 적용 방안

### 2. 데이터 시각화 (Data Visualization) 도입
현재 거의 모든 정보가 텍스트·숫자·뱃지로만 표현되어 있어 패턴 파악이 어려움. 아래 데이터에 대해 적합한 시각화 유형과 구현 방안을 제시:

- **리스크 온도**: 현재 텍스트 도트 → 반원형 게이지(Gauge) 또는 온도계 형태로 시각화
- **금리 커브**: 국고채 1/3/5/10/30년 수치를 수익률 곡선(Yield Curve) 라인 차트로 표현. 정상/평탄/역전 상태가 한눈에 보이도록
- **Track Record**: 승률·수익률을 도넛 차트(승/패 비율) + 수익률 바 차트로 시각화. 현재 한 줄 텍스트만으로는 성과 체감이 어려움
- **발굴 유형 분포**: 현재 CSS 가로 바 → 도넛/파이 차트 또는 트리맵으로 비율 직관성 강화
- **섹터 분포**: 태그 클라우드 → 섹터별 비중 원형 차트 또는 버블 차트
- **Top Picks 상승여력**: 숫자 나열 → 수평 불릿 차트(현재가 ↔ 목표가 범위)로 시각화
- **테마 신뢰도/확신도**: 숫자 → 원형 프로그레스 링 또는 별점 형태
- **전일 대비 변화량**: 핵심 지표(이슈 수, 테마 수 등)에 전일 대비 증감 화살표 + 스파크라인 (최근 7일 추이)

각 시각화에 대해:
  - 순수 CSS/SVG 인라인으로 구현하는 방안 (경량)
  - 차트 라이브러리(Chart.js 또는 lightweight 대안) 활용 방안
  - 두 방안의 트레이드오프 비교

### 3. 핵심 의사결정 정보의 시각적 강화
